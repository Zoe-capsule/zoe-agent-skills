# Evo Points - 积分赚取方法论

> 系统化提升项目权重，优化 Token 资源配置

---

## When to Use

✅ **使用场景：**
- 项目权重偏低，需要提升资源分配
- 日 Token 预算不足，需要优化使用效率
- 团队产出与回报不匹配，需要建立激励机制
- 需要系统化提升项目影响力和可见度

❌ **不适用场景：**
- 一次性任务，无需长期运营
- 项目即将结束，无需持续投入
- 团队没有持续产出能力

---

## Core Concepts

### 什么是积分？

在 OpenClaw 生态中：
```
积分 = 项目权重 = 资源分配权
```

| 权重等级 | 日 Token 预算 | 资源优先级 |
|---------|--------------|-----------|
| 高权重 (>40%) | 20,000+ | 优先模型、优先人员 |
| 中权重 (20-40%) | 10,000-20,000 | 标准配置 |
| 低权重 (<20%) | <10,000 | 基础配置 |

### 积分来源

| 来源 | 单次积分 | 月度潜力 | 难度 |
|-----|---------|---------|------|
| 任务完成 | 10-50 | 300-500 | ⭐ |
| 技术创新 | 50-200 | 200-400 | ⭐⭐ |
| 文档贡献 | 20-100 | 100-300 | ⭐ |
| 团队协作 | 10-30 | 50-150 | ⭐ |
| 成本节约 | 30-100 | 100-200 | ⭐⭐ |
| 外部贡献 | 50-300 | 100-500 | ⭐⭐⭐ |

## Quick Start

### 第一步：现状评估

```bash
# 检查当前状态
current_weight=26  # 当前权重百分比
current_budget=13000  # 当前日 Token 预算
target_weight=40  # 目标权重
target_budget=20000  # 目标日预算

echo "差距分析："
echo "权重需提升: $((target_weight - current_weight))%"
echo "预算需增加: $((target_budget - current_budget)) Token/日"
```

### 第二步：制定策略

**短期策略（1-3个月）：**
- 提升任务完成率和质量
- 建立标准化工作流程
- 增加文档输出

**中期策略（3-6个月）：**
- 技术创新和自动化
- 跨团队协作
- 外部影响力建设

### 第三步：执行与复盘

```
每日：15分钟复盘
每周：1小时深度总结
每月：半天进化日
```

## Best Practices

### 积分加速技巧

| 技巧 | 加成 | 说明 |
|-----|------|------|
| 提前1天完成 | +20% | 预留缓冲时间 |
| 提前3天完成 | +50% | 高效执行 |
| 超出预期质量 | +30% | 精益求精 |
| 附带额外价值 | +50% | 超预期交付 |

### 自我进化机制

**每日复盘三问：**
1. 今天完成了什么？（记录成果）
2. 遇到了什么问题？（记录困难）
3. 明天如何做得更好？（记录改进）

**进化指标：**
- 任务完成时间：3个月 -20%，6个月 -40%
- 自动化率：当前 20% → 3个月 40% → 6个月 60%
- 返工率：目标降低 50%

## Examples

### 示例 1：快速提升积分

**场景**：新项目启动，需要快速建立权重

**策略**：
```
Week 1-2: 集中完成高价值任务
  - 完成核心功能开发 (+200分)
  - 编写技术文档 (+50分)
  - 建立自动化流程 (+100分)

Week 3-4: 扩大影响力
  - 技术分享 (+80分)
  - 帮助其他项目 (+30分)
  - 优化成本 (+50分)

预期：首月 +500分
```

### 示例 2：持续优化

**场景**：成熟项目，追求长期增长

**策略**：
```
每月固定动作：
  - 10个高质任务 (+300分)
  - 2份技术文档 (+100分)
  - 1次技术创新 (+150分)
  - 团队协作 (+100分)

预期：月均 +650分，年 +7800分
```

| expected_points | number | 预期积分增长 |
